# LogHub Data Directory

This directory contains optional real-world log samples from the LogHub dataset.

## Adding LogHub Data (Optional)

The environment works with synthetic logs by default. To enhance real-world utility, you can optionally add LogHub samples:

### Supported LogHub Sources

- **HDFS**: Hadoop Distributed File System logs
- **BGL**: Blue Gene/L supercomputer logs
- **OpenStack**: OpenStack cloud platform logs
- **Apache**: Apache HTTP server logs

### Download Instructions

1. Visit the LogHub repository: https://github.com/logpai/loghub
2. Download sample logs from the datasets you want
3. Place them in this directory with the naming convention:
   - `hdfs_sample.log`
   - `bgl_sample.log`
   - `openstack_sample.log`
   - `apache_sample.log`

### Example: Adding HDFS Sample

```bash
# Download HDFS sample (approximately 1500 lines)
wget https://raw.githubusercontent.com/logpai/loghub/master/HDFS/HDFS_2k.log_structured.csv -O hdfs_sample.log

# Or manually download and rename
# The parser will automatically detect and parse the LogHub format
```

## Fallback Behavior

If LogHub files are not present, the environment automatically falls back to high-quality synthetic logs that simulate realistic anomaly patterns. This ensures the environment works out-of-the-box without requiring external data downloads.

## Data Source Selection

When resetting the environment, you can specify the data source:

```python
# Use synthetic logs (default)
env.reset(difficulty="easy", data_source="synthetic")

# Use LogHub HDFS logs (if hdfs_sample.log exists)
env.reset(difficulty="easy", data_source="loghub", log_source="HDFS")
```
